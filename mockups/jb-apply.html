📋 Dev Notes:
  • Route: /jobs/{id}/apply or /apply?jobId={id}
  • Step state managed client-side. Each step validates required fields before advancing — do not allow skipping forward.
  • Progress is auto-saved to localStorage keyed by jobId so the candidate can return later without losing data.
  • On final submit, POST to /api/public/applications. If the email already exists as a Candidate in the CRM, link the application to that record; otherwise create a new Candidate.
  • File uploads (resume, DD-214, certs) go to blob storage via POST /api/public/upload. Max 10MB per file, accept PDF/DOC/DOCX/JPG/PNG.
  • EEO demographic fields are optional and stored in a separate table — never joined to the application in any selection-related query.
  • Veteran's preference: if claimed, prompt for DD-214 upload in Step 3. Apply preference points per agency rules.
  • After submit: redirect to /apply/confirmation?ref={applicationId} with confirmation number. Send confirmation email to applicant.
  • "Save for Later" requires account creation (email + password). Show a modal prompt — not covered in this mockup.
  • The job banner and context header are populated from the same GET /api/public/jobs/{id} endpoint used by the detail page.
