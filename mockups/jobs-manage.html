Dev Notes:
  • Route: GET /jobs (CRM, authenticated)
  • Jobs scoped to org_id — agencies only see their own postings.
  • "View" opens the public-facing job detail page in a new tab (/jobs/{slug}).
  • "Candidates" count links to a job candidates view (not yet mocked up) — GET /jobs/{id}/candidates.
  • Pages column: A job has a 1:1 relationship with a page. If listing_page_type = "branded", the cell shows "Page Attached" linking to the web-builder page URL (/OrgSlug/PageSlug). Otherwise "—". The "View" action resolves to the branded page when one is attached, or the standard jb-detail view otherwise. Department-level pages are an org setting — not shown per-job in this table.
  • Job categories are global platform-level picklist values managed by platform admins only — individual agencies cannot add or modify categories. This keeps the public-facing job board filter consistent across all agencies.
  • Closing a job sets status = closed and closedAt = now(). The public listing immediately returns 404 or "Applications Closed" state.
  • "Reopen" sets status = active, resets closedAt, optionally prompts for new deadline.
  • Draft jobs are invisible on the public board until explicitly published.
  • Expired jobs (past deadline with no manual close) should auto-set to status = expired via a scheduled job.
