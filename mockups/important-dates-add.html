Before / After — Templates (100 MHS Interviews in a Month)
Today (without templates)
  1. Open Add Important Date
  2. Choose scope: Individual
  3. Pick Type: Interview
  4. Type Name: "Mental Health Specialist Interview"
  5. Search and select candidate
  6. Type Location: "450 W Main St…"
  7. Type Notes (45-min, license, references…)
  8. Set Start Date/Time
  9. Set End Date/Time
  10. Save
~10 interactions × 100 interviews = 1,000 actions/month.
Plus typo risk on every Notes and Location field.
With Templates
  1. Open Add Important Date
  2. Pick template "Interview: Mental Health Specialist"
  3. Search and select candidate
  4. Set Start Date/Time
  5. (End time auto-fills from template duration)
  6. Save
~5 interactions × 100 interviews = 500 actions/month.
Notes and location identical every time. Coordinator focuses on scheduling, not data entry.
📋 Dev Notes:
  • Scope toggle (Individual vs. Group) controls which invite section is shown. Default is Individual when opened from the global Calendar/toolbar; Individual with pre-selected candidate when opened from a candidate profile.
  • Individual scope: Candidate search field with autocomplete. Multiple candidates can be added (chips). No tag required.
  • Group scope: Candidate Association Tag dropdown drives roster membership via the workflow/tag system. Live count updates on tag change. Tag is optional — a group event can exist with no tag if candidates are added manually from the Event Detail page later.
  • Template dropdown fires GET to pre-fill all fields via JS (no full page reload). Date/time and candidate fields are intentionally NOT stored in templates.
  • New Tag Name is NOT stored in templates to prevent duplicate tags on every use.
  • "Save as Template" checkbox reveals template name input via JS toggle.
  • When the form is submitted, if a Candidate Association Tag is selected, the back-end should query all candidates with that tag and add them to the event's roster immediately (not lazily).
