Dev Notes — Job Board Listings (jb-listings.html)
  • Hosting: Co-hosted on the same Azure server as the AllStar Recruiter CRM. The job board is a public-facing section of the same application — served under a subdomain (e.g. jobs.allstarrecruiter.com) or a path prefix (e.g. /jobs). No separate deployment needed.
  • The job board uses no CRM app shell (no sidebar, no topbar) — it has its own public-facing navbar and footer while sharing the same backend, database, and Azure blob storage.
  • Job data is served from the same CRM database via public read-only routes — no auth required to browse. These routes are flagged as public in the API middleware and bypass the CRM auth guard.
  • Each job card links to /jobs/{id}. The router checks job.listing_page_type: if "branded", redirect to the attached web-builder page URL (/OrgSlug/PageSlug); if "standard", render the default job detail view.
  • Agency page: If the posting agency has a agency_page_url set in their org settings, the job card shows their logo avatar and an "Agency Page →" link. The logo is pulled from org.logoUrl (falls back to initials avatar).
  • Branded job page: Jobs with listing_page_type = "branded" show an iframe thumbnail preview of the attached web-builder page in the card (130×86px container, iframe scaled to 12.5%, pointer-events disabled). The card has a purple left border accent and "View Job →" routes to the web-builder page, not jb-detail.
  • Standard job (no pages): No logo, no pill, navy "View Job →" button, routes to standard jb-detail.
  • "Save Jobs" requires a candidate account — the login/register flow is not yet built. Clicking the link or bookmark icon should prompt sign-up/login.
  • Filters are reflected as URL query params (e.g. ?category=law-enforcement&location=austin,tx&radius=50&salary=50k) for shareable/bookmarkable URLs. Location accepts a city name or zip code; the backend geocodes it and filters by distance <= radius miles from the resolved coordinates.
  • "Post a Job" in the navbar links to the AllStar Recruiter CRM at /jobs/new (agency login required) — same server, no external redirect needed.
  • Category badge colors: Law Enforcement = blue, Fire Services = red, EMS = green, Corrections = amber, Dispatch = purple, Emergency Mgmt = sky.
  • Pagination assumes ~20 results per page; 247 results = 13 pages (show 12 as last for demo).
