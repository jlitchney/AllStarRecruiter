📋 Dev Notes:
  • Full page route — not a side panel. Route: /calendar/event/{id}
  • Server-side pagination + filtering. Must stay snappy at 5,000+ invitees.
  • Bulk bar appears only when ≥1 row is checked; "Select all on page" + "Select all 247" both needed.
  • Status lifecycle: Invited → Confirmed / Declined → (post-event) Attended / No-Show / Rescheduled. Marking a status applies the corresponding Association Tag to that candidate, which triggers any workflows listening for that tag.
  • Stat cards are clickable — clicking "No Response" filters the roster to that status.
  • Template badge links back to template management page.
  • Assigned Staff — many-to-many join (events ↔ users). The creator is auto-added as Organizer. All assigned staff can view the roster and record outcomes; only Organizers can edit event details or delete the event.
  • Registration Page — group events only. URL slug auto-generated from event name + date on save. QR code generated server-side or via library (e.g. qrcode.js). Self-registrations POST to the same endpoint as manual roster additions and apply the Invite Association Tag. The public-facing registration page is served by the All-Star Events web app (events-six-orpin.vercel.app).
