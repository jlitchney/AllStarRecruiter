Dev Notes:
  • Route: GET/POST /jobs/new, GET/PUT /jobs/{id}/edit
  • Job is scoped to org_id — agencies only manage their own postings.
  • On "Publish": set status = published, publishedAt = now(), make visible via GET /api/public/jobs.
  • Slug auto-generated from job title + org name + year on first publish: police-officer-entry-level-acme-pd-2026.
  • The public listing routes (/jobs/*) are served by the same Azure app but bypass CRM auth middleware.
  • Rich text fields (overview, requirements, qualifications, benefits) stored as sanitized HTML.
  • Selection process steps stored as ordered JSON array on the Job record.
  • Association Tags work identically to the Events module — applying a tag fires workflow automations.
  • "Linked Event" creates a join between Job and Event; applicants auto-added to the event roster with Invited status.
  • Max Applicants: when limit is reached, the public listing shows "Applications Closed" automatically.
